## 2. Основная концепция проекта

Project Viski — это **userspace сетевой стек**. 
- Мы не просто «проксируем» данные. 
- Мы забираем IP-пакеты из системного TUN-интерфейса Android.
- Передаем их в **gVisor (Netstack)** в Go-ядре.
- Внутри Go мы превращаем пакеты обратно в TCP-потоки, модифицируем их (фрагментация, обход DPI) и отправляем в сеть через обычные сокеты, которые «защищены» через `VpnService.protect()`.

## 3. Критически важные нюансы кода

### 3.1. Владение дескриптором (fdsan)
Никогда не используйте `parcelFd.fd` напрямую для передачи в Go. Используйте только `parcelFd.detachFd()`. В противном случае приложение упадет с ошибкой `fdsan` при первой же очистке памяти сборщиком мусора Go.

### 3.2. JNI Callback-и
Все логи и вызовы защиты сокетов идут из Go в Котлин через `bridge.c`. 
- Используется динамическая регистрация (`JNI_OnLoad`).
- Callback-и должны быть потокобезопасными и проверять наличие исключений в JVM (`ExceptionCheck`).

### 3.3. Контекст остановки
Для остановки ядра используется `context.WithCancel`. Просто «закрыть стек» недостаточно — нужно отменить контекст, чтобы прервать все активные `io.Copy` в горутинах прокси.

## 4. Как начать разработку (Epic 3)

Если ваша задача — реализовать обход DPI:
1. Изучите **[Фрагментация на Android.md](./Фрагментация%20на%20Android.md)**.
2. Логика модификации TCP-сегментов должна располагаться в `go-core/stack/proxy.go` перед вызовом `Dial`.
3. Параметры фрагментации должны приходить из `main.go` через JSON-конфиг.
