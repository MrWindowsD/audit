# Codemap — Project Viski (Network Optimizer)

Актуальная структура проекта и описание архитектурных решений после завершения **стабилизации Epic 2**.

---

## 1. Назначение продукта

Локальный «оптимизатор сети» на базе `VpnService` с Go-ядром. Использует **gVisor Netstack** для обработки трафика в userspace и **Transparent Proxy** для управления соединениями.

## 3. Архитектурные изменения и исправления (Стабилизация)

### 3.1. Управление жизненным циклом (The "Off Switch")
- **Проблема:** Ядро продолжало работать после остановки сервиса, вызывая зависания и утечки.
- **Решение:** Внедрен `context.Context` во все прокси-обработчики Go. Метод `StopEngine` теперь мгновенно отменяет контекст, разрывая все активные TCP/UDP сессии.
- **UI:** В `MainActivity.kt` добавлена кнопка остановки и индикатор «Обработка...», связанный с состоянием `isWaiting` в сервисе.

### 3.2. Решение проблемы `fdsan` (Критично)
- **Проблема:** Краш при сворачивании приложения с ошибкой `fdsan: attempted to close file descriptor... owned by ParcelFileDescriptor`.
- **Причина:** GC в Go закрывал `tunFd` через финализатор `os.File`, пока Java-объект еще считал себя владельцем дескриптора.
- **Решение:** 
  1. В Kotlin применен `parcelFd.detachFd()`, передающий полное владение дескриптором в Go.
  2. В Go структура `Engine` теперь удерживает `tunFile *os.File` на протяжении всего времени работы, предотвращая преждевременное закрытие.

### 3.3. Рефакторинг JNI (Dynamic Registration)
- **bridge.c:** Переведен на `JNI_OnLoad` с использованием `RegisterNatives`. Это ускорило вызов методов и убрало зависимость от длинных имен функций.
- **Безопасность:** Кэширование `jmethodID` при загрузке предотвращает краши при вызове callback-ов (`protectSocket`, `logFromGo`) во время завершения работы приложения.

## 4. Структура репозитория

```
ProjectViski/
├── app/
│   └── src/main/
│       ├── java/com/mrwindowsd/optimizer/
│       │   ├── MainActivity.kt           # Реактивный UI (Start/Stop, Loading)
│       │   ├── core/CoreBridge.kt         # JNI мост, статические callback-и
│       │   └── service/
│       │       └── OptimizerVpnService.kt # Управление TUN (Detached FD), жизненный цикл
├── go-core/
│   ├── main.go                 # Точка входа Go, парсинг JSON-конфига, Mutex-защита синглтона Engine
│   ├── bridge.c                # Dynamic JNI, кэширование методов, callback-и с защитой от паник
│   ├── build.ps1               # Кросс-сборка (fix кодировок и синтаксиса)
│   └── stack/
│       ├── netstack.go         # Стек gVisor, Promiscuous Mode, управление жизненным циклом FD
│       └── proxy.go            # TCP/UDP проксирование с поддержкой Context Cancellation
└── docs/                       # Единый центр документации
```
