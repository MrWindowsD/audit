# Release Notes - Project Viski

Этот документ содержит хронологию изменений, исправлений и улучшений проекта. Он служит контекстом для разработчиков и ИИ-ассистентов, чтобы понимать, почему были приняты те или иные решения.

---

## [Release #3.4] - SNI Parser & Logic Fixes (Хирургическая точность)

### 🚀 Исправления и улучшения
- **Parser Fix:** Исправлена критическая ошибка использования `cryptobyte` в `mixed_case.go`. Теперь тело Handshake читается из родительской строки, что восстанавливает цепочку парсинга ClientHello.
- **Header Offset:** Скорректирован оффсет TLS Record Header (5 байт). Теперь Handshake Type определяется безошибочно.
- **Resilient Handshake:** Флаг `firstDone` в `MixedCaseMiddleware` теперь выставляется только при успешной мутации или окончательном отказе. Это дает системе шанс поймать ClientHello, если он фрагментирован на уровне приложения.
- **Diagnostic Logging:** Добавлены детальные логи для отслеживания первого `Write` и результатов поиска SNI.

## [Release #3.3] - Comprehensive Audit & Stability Polish

### 🚀 Улучшения и фиксы
- **Resource Management:** Исправлена утечка файлового дескриптора TUN в `main.go` при ошибках инициализации.
- **Connection Reliability:** Пересмотрена логика закрытия сокетов в `proxy.go`. Теперь очистка гарантирована даже при сбоях в `Pipeline`.
- **Auto-Tuner Optimization:** Улучшена работа с контекстами в `fragmentation.go`, исключены утечки горутин при таймаутах раундов.
- **Log Cleanup:** Оптимизировано логирование в `proxy.go`, ошибки `io.EOF` больше не помечаются как критические.

## [Release #3.2] - Stability & Bypass (Броня и Очистка)

### 🚀 Новые возможности
- **Anti-Crash Protection:** Внедрена система `recover()` во всех критических горутинах Go-ядра.
    - **Auto-Tuner:** Защита горутин параллельного пробирования в `fragmentation.go`.
    - **Proxy TCP/UDP:** Защита основных и вспомогательных горутин (копирование трафика, управление контекстом) в `proxy.go`.
- **Dynamic Test Domains:** Список доменов для тюнинга теперь передается из Kotlin (JSON).
- **Silent Local Bypass:** 
    - Мгновенное отклонение (TCP RST) запросов на локальные IP (10.x, 192.x, 127.x) прямо в Netstack.
    - Игнорирование локальных UDP пакетов.
    - Устранение таймаутов и лишнего мусора в логах при обращении системы к локальным ресурсам.
- **Стабильность:** Предотвращение `Fatal CGO error` и падения всего Android-приложения при возникновении паники в асинхронном Go-коде.

## [Release #3.1] - Epic 3 Expansion: Продвинутая фрагментация (В работе)

### 🚀 Планируемые возможности (Цель: 85-90% эффективности)
- **TTL Trick:** Внедрение манипуляций с полем Time To Live для обмана DPI без доставки пакетов на сервер.
- **Fake TCP Packets:** Инъекция "мусорных" пакетов с невалидными параметрами (Checksums/Seq) для десинхронизации парсеров провайдера.
- **HTTP Header Obfuscation:** Манипуляция регистром (hOsT) и лишними пробелами в незашифрованном трафике.
- **Combined Strategies:** Автоматическая генерация профилей, сочетающих L4, L7 и TTL-трюки для максимального пробития.

## [Release #3] - Epic 3: Фрагментация и Auto-Tuner (Стабильный базис)

### 🚀 Новые возможности
- **Auto-Tuner (Deep Turbo Probing):** 
    - Параллельный пробинг доменов с ограничением (Semaphore = 5).
    - Динамические таймауты раундов (от 7 до 15 секунд) в зависимости от агрессивности стратегии.
    - Отправка статуса "квадратиков" в реальном времени через JNI `sendTuningStatus`.
    - Ранний выход при достижении эффективности 90%+.
- **Bypass локальных сетей:** 
    - Автоматическое определение локальных IP (10.x, 192.x, 127.x).
    - Трафик на локальные ресурсы идет напрямую, минуя фрагментацию (исправляет доступ к 10.0.0.1).
- **Персистентность конфига:**
    - `ConfigManager` (Kotlin) сохраняет найденный профиль в `SharedPreferences`.
    - При повторном запуске тюнинг пропускается, если профиль уже есть.
- **Матрица стратегий:** Внедрена официальная "железобетонная" матрица из 6 профилей (None, L7 x2, L4 x3).

### 🛠 Исправления и оптимизации
- **Cleanup `main.go`:** Удалены дубликаты функций, исправлена структура JSON-конфига.
- **Engine API:** Добавлен метод `Context()` в `stack.Engine` для корректного управления жизненным циклом тюнера.
- **JNI Bridge:** Исправлена инициализация `ConfigManager` в `MainActivity` для предотвращения `UninitializedPropertyAccessException`.
- **Debug Logging:** В Go-ядре добавлен вывод входящего JSON-конфига для отладки связи Kotlin -> Go.

## 📝 Инструкция для будущих правок ИИ

При внесении значимых изменений в проект:
1. **НЕ ПЕРЕПИСЫВАЙТЕ** весь CODEMAP или README, если это не требуется архитектурно.
2. **ДОБАВЬТЕ** новый блок в начало `RELEASE_NOTES.md` под заголовком `[Release #X]`.
3. Опишите:
    - **🚀 Новые возможности**: что добавлено.
    - **🛠 Исправления**: что пофикшено.
    - **⚠️ Изменения API**: если изменились JNI-методы или структура конфига.
4. **ОБНОВИТЕ** статус эпиков в `README.md` и `CODEMAP.md` (например, с `🔄 Next` на `✅ Done`).
5. **ПРОВЕРЬТЕ**, чтобы новые файлы были отражены в структуре проекта в `CODEMAP.md`.
