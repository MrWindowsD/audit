# Project Viski — «Оптимизатор сети»

Это основной индекс документации проекта. Здесь описаны архитектура, роадмап и текущий статус разработки.

## 🚀 Быстрый старт (Сборка)

1. **Требования:** JDK 17, Android SDK, NDK (`30.0.14904198`), Go `1.26+`.
2. **Сборка Go-ядра:** В папке `go-core` выполните `.\build.ps1`. Библиотеки `libcore.so` будут разложены по ABI в `app/src/main/jniLibs/`.
3. **Сборка APK:** Выполните `.\gradlew.bat assembleDebug`.

## ⚖️ Принципы (обязательны для кода и ревью)

1. **Фрагментация не ломает связность.** Если с активным профилем приложения перестают открываться сайты/сервисы, которые без VPN работают — **стратегия или пайплайн бракованы**, а не «так надо». Epic 3: один глобальный профиль на весь TCP (кроме loopback), без whitelist из `test_domains`.
2. **Split-tunnel — отдельный Epic, не сейчас.** Выборочная маршрутизация по **доменам** и **приложениям** (rule-sets, `addDisallowedApplication`, route-options) — **следующий этап**, не костыль в Auto-Tuner. Запрещено: привязывать live-трафик к `test_domains`, Sniper, «табуретка» из списка тестов.

## 🛠 Ключевые достижения текущего билда

*   **Умная фрагментация:** Автоматический подбор профиля (L7/L4) под провайдера.
*   **Параллельный тюнинг:** Глубокое сканирование сети за 30-60 секунд.
*   **Локальный Bypass:** Корректная работа с локальными сетями и роутерами (10.0.0.1).
*   **Персистентность:** Сохранение найденных настроек в SharedPreferences.
*   **Стабильность:** Решена проблема `fdsan` и реализована корректная остановка горутин.
